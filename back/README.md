Snippet Manager API
Бекенд-частина додатку для керування фрагментами коду (сніпетами). Побудовано на NestJS із використанням MongoDB (Mongoose).

🚀 Як запустити локально
1. Попередні вимоги
Переконайтеся, що у вас встановлені:

Node.js (версія 18 або новіша)

npm або yarn

MongoDB (локально або рядок підключення до Atlas)

2. Встановлення залежностей
Bash
npm install
3. Налаштування змінних оточення
Створіть файл .env у корені проекту та скопіюйте туди дані з прикладу нижче.

4. Запуск у режимі розробки
Bash
npm run start:dev
Додаток буде доступний за адресою: http://localhost:3001 (або ваш PORT).

⚙️ Змінні оточення (.env.example)
Створіть файл .env.example, щоб інші розробники знали, які ключі потрібні:

Метод,Ендпоінт,Опис
GET ,/snippets, "Отримати всі сніпети (фільтри: q, tag, page)"
GET, /snippets/tags, Отримати список усіх унікальних тегів
GET, /snippets/:id, Отримати конкретний сніпет за ID
POST, /snippets, Створити новий сніпет
PATCH, /snippets/:id, Оновити існуючий сніпет
DELETE, /snippets/:id, Видалити сніпет

Білд та Production режим
Якщо ви готові деплоїти додаток:

1. Компіляція (Build)
Ця команда перетворює TypeScript-код у чистий JavaScript у папку /dist.

Bash
npm run build

2. Запуск у прод-режимі
Запуск уже скомпільованого коду:

Bash
npm run start:prod
