Snippet Manager Frontend
Клієнтська частина додатку для керування фрагментами коду. Побудовано на Next.js (App Router) з використанням TanStack Query, Axios та Tailwind CSS.

🚀 Як запустити локально
1. Попередні вимоги
Переконайтеся, що у вас встановлені:

Node.js (версія 18.17 або новіша)

npm або pnpm / yarn

2. Встановлення залежностей
Виконайте команду в корені папки фронтенду:

Bash
npm install
3. Налаштування змінних оточення
Створіть файл .env у корені проекту. Ви можете скопіювати вміст із прикладу нижче.

Важливо: Не забудьте вказати правильний URL вашого бекенду.

4. Запуск у режимі розробки
Bash
npm run dev

Відкрийте http://localhost:3000 у браузері, щоб побачити результат.

⚙️ Змінні оточення (.env.example)
Для коректної роботи фронтенду потрібно вказати адресу API. У Next.js змінні, які мають бути доступні в браузері, повинні починатися з NEXT_PUBLIC_.

Фрагмент кода
# URL вашого бекенд API
NEXT_PUBLIC_API_URL=http://localhost:4000

Білд та Production режим
Для підготовки проекту до розгортання на сервері (наприклад, Vercel або власний VPS):

1. Збірка (Build)
Next.js оптимізує код, проводить статичну генерацію та готує проект до продакшну.

Bash
npm run build
2. Запуск (Start)
Запуск сервера в режимі продакшн (після успішного білду):

Bash
npm run start
